Today  for the very first time I played with github.For all of those who do not have any idea about what github is and what it offers to us, i would say that github is a platform wherein we can upload/share our files irrespective of their types. To be more technical on it one can say its a Version Control tool which provides any individual a way to upload files to it so that anyone else who wants to see/update the file can just open the file, have a look, make changes if necessary and commit them.I have used IBM Clearcase before and i found that very clumsy to use, But here in GIT things are quite clear and easy to use,UI looks light and attractive.

GIT Basics
As soon as a file is changed a notification is sent to the owner of the file with the information like who changed the file, when it was changed and a description about the change.

If the owner wants to accept the changes , he/she can merge the changed file and the original file or  the changes can be denied too.This is a bit tedious as in a team for every change the owner has to do a set of actions to approve the changes and merge the file.

To overcome this process we have a way in which the repository owner can add all his team mates as collaborators. This gives all the collaborators a right to change any file and push it back to the repository. This is a simple process and saves time.

here is how we add collaborators:

click on the admin button of a Repository and then click on collaborator.On the drop down list type the persons name whom you want to add as a collaborator.

Once we are done with adding the collaborators, all our folks who are added will have the read/write access to that repository.
