HTML Box Model


The box model in CSS describes the boxes which are being generated for HTML-elements. The box model also contains detailed options regarding adjusting margin, border, padding and content for each element. Here is the diagram for a typical Box Model:



As we can see, a box is made up of four distinct parts.The margin which is completely invisible. It has no background color, and will not obstruct elements behind it. Second one is the border. The border outlines the visible portion of the element. Inside the border is the third part of the box, the padding, and then inside that the content area of the box. The padding defines the space between the content area of the box and the border.

To calculate the overall width of a box, including all padding, borders and margins, we can use the formula i.e.
*total box width=content area width+ left padding+right padding+left border+right border+left margin+right margin.

Before i knew this box model I always used to get confused with these margin ,padding etc and it was irritating to do these operations as i never knew where the padding will be added.Now as I am aware of this box model I can perform all that operations fast and effectively.


Must know HTML TAGS

Here are few HTML Tags that i think every Blogger/developer must know:

  1. Anchor Link Tag -<a href="…">link</a>
  2. Image Tag- <img src="…" />
  3. Headings -<h2>, <h3>, <h4>, <h5>
  4. Paragraph- <p>Text here</p>
  5. Bold -<strong> not <b>
  6. Italic -<em> not <i>
  7. Lists -<ol> and <ul>
  8. Blockquote -<blockquote> and <cite>
  9. Line Break -<br />
  10. Horizontal Line -<hr />
  11. Preformatted -<pre>

HTML Best Practices                                       


1)Always Close your tags: Its one of the most important practice that every developer should follow.Leaving tags open may lead to some validation and glitch issues at every turn.


2)Place the JavaScript file at the bottom:the primary goal is to make the page load as quickly as possible for the user. When loading a script, the browser can’t continue on until the entire file has been loaded. So the user will have to wait longer before noticing any progress.If we have JavaScript files whose only purpose is to add functionality — for example, after a button is clicked —we should place them at the bottom, just before the closing body tag. This is absolutely a best practice.


3)All images must have the 'Alt' tag:The required alt attribute specifies an alternate text for an image, if the image cannot be displayed.The alt attribute provides alternative information for an image if a user for some reason cannot view it (because of slow connection, an error in the src attribute, or if the user uses a screen reader).
  
4)Always declare a Doctype: The doctype declaration should be the first thing in the HTML documents. The doctype declaration tells the browser about the XHTML standards we will be using and helps it read and render the markup correctly.

5)Separate content from presentation:HTML is the content, CSS provides content a visual presentation.These two should never be mixed. Always a separate CSS should be created for the styles. This will help us and future developers that might work on our code and make changes to the design

6)Markups should be in Lower Case:It is an industry-standard practice to keep the markup lower-cased. Keeping it in the uppercase will surely not affect the page rendering or functionality but it will surely affect the readability of the HTML code and so it should be avoided.
